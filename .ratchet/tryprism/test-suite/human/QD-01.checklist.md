## Desktop Review

### Overall Aesthetic

- [ ] **Dark background feels intentional** — not just a CSS `background: black`. Backgrounds should use deep, slightly-tinted darks (e.g., `#0a0a0f`, `#1a1a2e` range or similar), creating depth. Flat black feels cheap.
- [ ] **Color palette is cohesive** — accent colors from the prismatic spectrum (not random or clashing). Each of the 9 Enneagram types should have a unique, tasteful accent color.
- [ ] **No bright or neon colors** — in dark mode, saturated neons (hot pink, electric blue, etc.) break the premium/calm aesthetic. Accent colors should be luminous but not garish.
- [ ] **Typography is professional** — headings use an appropriate typeface (serif or refined sans-serif), body text is legible (min 16px), line-height is generous (1.5–1.8).
- [ ] **Heading hierarchy is clear** — visually distinct sizes/weights for h1, h2, h3. Not everything the same size.
- [ ] **Generous whitespace** — sections are not cramped. Content breathes. The app doesn't feel like it's trying to fit everything above the fold.
- [ ] **No visual clutter** — the interface shows only what's needed. No unnecessary decorative elements, badges, or UI chrome.

### Home Page

- [ ] **Logo/branding is refined** — the "Prism" identity is expressed visually (light-refraction metaphor, spectrum, etc.). Not just text.
- [ ] **Mode cards (quick/full) are visually distinct and attractive** — clear visual hierarchy between them. Hover/focus states are polished.
- [ ] **Language and theme toggles are subtle but accessible** — not oversized or distracting, but clearly clickable.
- [ ] **The page communicates what the app is without reading a manual** — a first-time visitor should immediately understand this is a personality test.

### Quiz Page

- [ ] **Question text is comfortable to read** — proper font size, good contrast, not straining.
- [ ] **Likert scale options are visually clear** — the selected state is obvious (not just a faint border change). Scale endpoints (Strongly Disagree / Strongly Agree) are labeled.
- [ ] **Progress bar is aesthetically integrated** — not a plain browser-default progress element. Animated if applicable.
- [ ] **Navigation buttons (back/next) are appropriately sized and styled** — not generic browser-default buttons.
- [ ] **No visual anxiety or time pressure** — the design feels calm throughout the quiz.

### Results Page

- [ ] **Primary type presentation is the hero moment** — the type number, name, and brief tagline should be visually striking — this is the "reveal" the user has been building toward.
- [ ] **Wing indicator is elegantly displayed** — "4w5" style notation or equivalent, styled appropriately.
- [ ] **Score chart/radar is beautiful** — not just a default Chart.js/D3 render with no customization. Colors should correspond to the type's accent color. Clean gridlines (or no gridlines).
- [ ] **Growth and stress arrows are visually intuitive** — the diagram communicates direction without requiring a legend read.
- [ ] **Type description text is pleasant to read** — appropriate line-length (max 70–80 characters per line on desktop), good leading.
- [ ] **The page doesn't feel like a wall of text** — sections are visually separated, key numbers/insights are called out with visual weight.

### Animations and Transitions

- [ ] **Page transitions are smooth** — no jarring instant switches between home/quiz/results.
- [ ] **Progress bar update is animated** — doesn't jump instantly.
- [ ] **Answer selection has a micro-interaction** — selecting a Likert option has satisfying feedback (color change, scale, ripple, etc.).
- [ ] **Animations are subtle, not flashy** — 200–400ms transitions that add polish without distracting.

## Mobile Review (375px width — iPhone SE / small phone)

- [ ] **No horizontal scrollbar** — the entire app fits within the 375px viewport.
- [ ] **Touch targets are at least 44x44px** — buttons, answer options, toggles are tappable without precision.
- [ ] **Text is readable without zooming** — minimum 16px font on small screen.
- [ ] **Mode cards stack vertically** — the home page doesn't try to show cards side-by-side on small screen.
- [ ] **Quiz layout is comfortable** — questions and Likert options don't feel cramped.
- [ ] **Results page scrolls naturally** — all sections accessible by scrolling, nothing hidden or clipped.

## Specific Design Anti-Pattern Checks

Verify the following are NOT present:

- [ ] No Bootstrap, Material UI, or Ant Design default components — this should not look like a template.
- [ ] No gamification elements — no points, badges, streaks, confetti.
- [ ] No countdown timer or time pressure indicators.
- [ ] No social proof copy ("X people have taken this test today").
- [ ] No login/signup prompts or walls.
- [ ] No ads, affiliate links, or upselling.

## Scoring

After completing the checklist, assign a score:

- **5** — Nearly all checklist items pass, the design genuinely impresses, premium feel is strong.
- **4** — Most items pass (≥85%), minor rough edges that wouldn't prevent shipping.
- **3** — About 70% of items pass, design is functional but not distinctive.
- **2** — 50–70% pass, noticeable visual problems that affect the experience.
- **1** — Below 50%, the design needs significant rework.
